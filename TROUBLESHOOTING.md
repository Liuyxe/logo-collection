# 🔧 跨浏览器同步故障排除指南

## ❓ 问题：不同浏览器内容不一样

### 可能原因

1. **未配置GitHub同步**
   - 首次使用需要配置GitHub信息
   - 配置后才能实现跨浏览器同步

2. **未点击同步按钮**
   - 上传logo后需要手动点击"🔄 同步数据"按钮
   - 切换浏览器后需要点击同步加载最新数据

3. **GitHub配置错误**
   - Token可能过期或无效
   - 用户名或仓库名称不正确
   - Token没有足够的权限

4. **网络问题**
   - 无法连接到GitHub API
   - 网络超时或连接中断

5. **数据未正确同步**
   - 上传后忘记点击同步
   - GitHub Issues中没有对应数据

---

## 🔍 诊断步骤

### 步骤1：检查配置

1. 打开网站
2. 点击"🔍 调试"按钮
3. 查看"GitHub配置"部分：
   - 如果显示"未配置"，需要先配置GitHub信息
   - 如果显示"已配置"，检查用户名、仓库名称和Token是否正确

### 步骤2：检查数据

在调试面板中查看：
- **本地logo数量**: 应该大于0（如果上传了logo）
- **存储状态**: 所有localStorage项应该显示"存在"
- **最后上传**: 应该显示最近上传的时间

### 步骤3：测试同步

1. 上传一个测试logo
2. 点击"🔄 同步数据"按钮
3. 观察右上角的同步状态：
   - ✅ 绿色 = 同步成功
   - ❌ 红色 = 同步失败
   - 🔵 蓝色 = 正在同步

### 步骤4：检查浏览器控制台

1. 按F12打开开发者工具
2. 切换到"Console"标签
3. 查看是否有错误信息：
   - 红色错误表示有问题
   - 蓝色信息表示正常操作

## 🛠️ 常见问题解决

### 问题1：同步失败 - 401 Unauthorized

**原因**: Token无效或过期

**解决方法**:
1. 访问 https://github.com/settings/tokens
2. 删除旧的Token
3. 创建新的Token
4. 在网站中点击"🔄 同步数据"按钮
5. 更新Token并保存

### 问题2：同步失败 - 404 Not Found

**原因**: 仓库名称或用户名不正确

**解决方法**:
1. 检查GitHub用户名是否正确
2. 检查仓库名称是否正确（区分大小写）
3. 确认仓库是否存在
4. 重新配置GitHub信息

### 问题3：同步失败 - 403 Forbidden

**原因**: Token没有足够的权限

**解决方法**:
1. 确保创建Token时勾选了 `repo` 权限
2. 如果仓库是公开的，勾选 `public_repo` 权限
3. 重新生成Token并更新配置

### 问题4：同步失败 - 422 Unprocessable Entity

**原因**: Issue标题或内容格式错误

**解决方法**:
1. 检查logo名称是否包含特殊字符
2. 确保图片Base64编码正确
3. 尝试删除有问题的logo后重新上传

### 问题5：加载GitHub数据失败

**原因**: 网络问题或GitHub API限制

**解决方法**:
1. 检查网络连接
2. 等待几分钟后重试
3. 检查是否超过GitHub API限制（每小时5000次请求）

### 问题6：GitHub中没有数据

**原因**: 首次使用或数据被删除

**解决方法**:
1. 上传至少一个logo
2. 点击"🔄 同步数据"按钮
3. 确认数据已保存到GitHub Issues
4. 检查GitHub仓库的Issues页面

### 问题7：数据不一致

**原因**: 不同步或同步失败

**解决方法**:
1. 在所有浏览器中点击"🔄 同步数据"按钮
2. 等待同步完成
3. 刷新页面确认数据一致
4. 如果仍然不一致，清除localStorage后重新同步

## 🔄 重置步骤

### 清除所有数据

如果问题持续，可以尝试清除所有数据：

1. 打开浏览器控制台（F12）
2. 输入以下命令：
   ```javascript
   localStorage.clear();
   location.reload();
   ```
3. 按回车执行
4. 页面会刷新，所有数据被清除
5. 重新配置GitHub并同步

### 只清除GitHub配置

如果只想重新配置GitHub：

1. 打开浏览器控制台（F12）
2. 输入以下命令：
   ```javascript
   localStorage.removeItem('githubConfig');
   location.reload();
   ```
3. 按回车执行
4. 页面会刷新，GitHub配置被清除
5. 重新配置GitHub信息

## 💡 最佳实践

### 1. 定期同步

- 每次上传logo后立即同步
- 切换浏览器前先同步
- 定期（如每周）点击同步确保数据一致

### 2. 备份数据

- GitHub Issues本身就是备份
- 可以随时在GitHub查看历史数据
- 即使本地数据丢失也能从GitHub恢复

### 3. 监控同步状态

- 注意右上角的同步状态提示
- 如果出现红色错误，立即检查
- 查看浏览器控制台的详细错误信息

### 4. 使用调试功能

- 点击"🔍 调试"按钮查看详细信息
- 对比不同浏览器的调试信息
- 确认数据是否正确同步
